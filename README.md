# Multimodal Fashion & Context Retrieval

> **Note:** Due to GitHub's 100 MB file size limit, the required models, datasets, and generated retrieval files are hosted as public Kaggle datasets. Please add the required datasets to your Kaggle notebook before running the project.

---

# Running the Indexer

The Indexer generates the CLIP embeddings, metadata, and FAISS index required for image retrieval.

## Step 1: Add the following Kaggle datasets to your notebook

### 1. CLIP Embeddings
https://www.kaggle.com/datasets/riddhipandya1/clip-embeddings

*Provides the pretrained OpenCLIP embedding model.*

### 2. CLIP Tensors
https://www.kaggle.com/datasets/riddhipandya1/clip-tensors-new

*Contains the pretrained OpenCLIP model weights.*

### 3. Fashionpedia Images
https://www.kaggle.com/datasets/riddhipandya1/images

*Contains the Fashionpedia test images used for indexing.*

### 4. Florence-2 Model
https://www.kaggle.com/datasets/riddhipandya1/florence-official

*Provides the Florence-2 Vision Language Model used for dense caption generation.*

---

## Step 2

Open and run:

```
1_Indexer.ipynb
```

This generates:

- `clip_embeddings.npy`
- `faiss.index`
- `fashion_metadata_1500.json`
- `inventory.csv`

---

# Running the Retriever

The Retriever performs multimodal image retrieval using the precomputed index generated by the Indexer.

## Step 1: Add the following Kaggle datasets

### 1. CLIP Tensors
https://www.kaggle.com/datasets/riddhipandya1/clip-tensors-neww

*Contains the pretrained OpenCLIP model weights.*

### 2. Fashionpedia Images
https://www.kaggle.com/datasets/riddhipandya1/images

*Contains the Fashionpedia test images used for retrieval.*

### 3. Indexer Results
https://www.kaggle.com/datasets/riddhipandya1/results

*Contains the generated FAISS index, metadata, embeddings, and inventory.*

---

## Step 2

Open and run:

```
2_Retriever.ipynb
```

The notebook loads the precomputed index and performs multimodal fashion image retrieval.
